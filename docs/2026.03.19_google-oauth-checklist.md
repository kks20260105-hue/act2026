# Google OAuth 로그인 점검 체크리스트

## 1. Google Cloud Console 설정

| 항목 | 값 |
|------|-----|
| API 유형 | OAuth 2.0 클라이언트 ID (웹 애플리케이션) |
| 승인된 JavaScript 출처 | `http://localhost:3000` |
| 승인된 JavaScript 출처 (운영) | `https://act2026.vercel.app` |
| 승인된 리디렉션 URI (로컬) | `http://localhost:4000/api/auth/google-callback` |
| 승인된 리디렉션 URI (운영) | `https://act2026.vercel.app/api/auth/google-callback` |

> ⚠️ URI는 `/callback` 아닌 `-callback` 입니다 (슬래시 아닌 하이픈)

## 2. GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET / GOOGLE_CALLBACK_URL 확인 방법

### Step 1. Google Cloud Console 접속
1. https://console.cloud.google.com 접속 (Google 계정 로그인)
2. 상단 프로젝트 선택기 클릭 → **새 프로젝트** 생성 (또는 기존 프로젝트 선택)
   - 프로젝트 이름 예시: `act2026-portal`

### Step 2. OAuth 동의 화면 설정
1. 좌측 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
2. User Type: **외부** 선택 → 만들기
3. 앱 이름, 지원 이메일 입력 후 저장
4. 테스트 사용자에 본인 Google 계정 추가 (외부 앱은 게시 전까지 테스트 사용자만 로그인 가능)

### Step 3. OAuth 2.0 클라이언트 ID 생성
1. 좌측 메뉴 → **API 및 서비스** → **사용자 인증 정보**
2. 상단 **+ 사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
3. 애플리케이션 유형: **웹 애플리케이션**
4. 이름: `act2026-web` (임의 지정)
5. **승인된 JavaScript 출처** 추가:
   ```
   http://localhost:3000
   https://act2026.vercel.app
   ```
6. **승인된 리디렉션 URI** 추가:
   ```
   http://localhost:4000/api/auth/google-callback
   https://act2026.vercel.app/api/auth/google-callback
   ```
7. **만들기** 클릭

### Step 4. 값 확인
생성 완료 팝업 또는 **사용자 인증 정보** 목록에서 해당 클라이언트 클릭 시 확인:

| 항목 | 위치 |
|------|------|
| `GOOGLE_CLIENT_ID` | **클라이언트 ID** 항목 (숫자-숫자.apps.googleusercontent.com 형식) |
| `GOOGLE_CLIENT_SECRET` | **클라이언트 보안 비밀번호** 항목 (GOCSPX-... 형식) |
| `GOOGLE_CALLBACK_URL` | 위에서 직접 입력한 URI (환경에 따라 두 개 중 선택) |

---

### 환경변수 적용

**로컬 (`backend/.env`)**
```dotenv
GOOGLE_CLIENT_ID=1234567890-abcdefg.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxxxxxxxxxxxxxxxxxxx
GOOGLE_CALLBACK_URL=http://localhost:4000/api/auth/google-callback
```

## 3. 로그인 흐름 점검

```
[프론트] /api/auth/google 클릭
    ↓
[Vercel/API] state 생성 → google_oauth_state 쿠키 저장 → accounts.google.com 리다이렉트
    ↓
[Google] 사용자 동의 → /api/auth/google-callback?code=xxx&state=yyy 으로 복귀
    ↓
[Vercel/API] state 검증 → 토큰 교환 (googleapis.com/token)
    ↓
[Vercel/API] 프로필 조회 (googleapis.com/oauth2/v2/userinfo)
    ↓
[Supabase] uf_upsert_social_user → name + display_name 저장
    ↓
[Vercel/API] JWT 발급 → /oauth/callback?token=...&provider=Google 리다이렉트
    ↓
[프론트] JWT 파싱 → 로그인 완료
```

## 4. 오류 코드별 원인 및 해결책

| URL 오류 파라미터 | 원인 | 해결 |
|-------------------|------|------|
| `?error=google_denied` | 사용자 동의 취소 또는 `code` 누락 | 다시 로그인 시도 |
| `?error=google_state` | CSRF state 불일치 (쿠키 만료 5분) | 다시 로그인 시도 |
| `?error=google_config` | `GOOGLE_CLIENT_ID` or `SECRET` 환경변수 미설정 | .env 또는 Vercel 환경변수 확인 |
| `?error=google_token` | 토큰 교환 실패 | Redirect URI가 Google Console 등록값과 정확히 일치하는지 확인 |
| `?error=google_profile` | userinfo API 호출 실패 | access_token 유효 여부, Google API 활성화 여부 확인 |
| `?error=google_db` | Supabase upsert 실패 | DB 마이그레이션(20260318) 적용 여부, SUPABASE_SERVICE_KEY 확인 |
| `?error=google_server` | 서버 예외 | Vercel 함수 로그 확인 |

## 5. 로그인 후 DB 확인 쿼리

```sql
-- Google 사용자 최근 로그인 확인
SELECT id, email, name, display_name, profile_image, updated_at
FROM public.users
WHERE provider = 'google'
ORDER BY updated_at DESC NULLS LAST
LIMIT 10;

-- display_name 누락 확인
SELECT COUNT(*) AS missing
FROM public.users
WHERE provider = 'google'
  AND (display_name IS NULL OR btrim(display_name) = '');
```
