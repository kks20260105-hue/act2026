## SB-001. 포털 메인 홈

- **화면 설명**: 로그인 후 최초 진입하는 포털 메인 화면. GNB·LNB·컨텐츠 영역으로 구성되며, 로그인 상태에 따라 메뉴 노출이 달라진다.
- **접근 가능 Role**: USER, MANAGER, ADMIN, SUPER_ADMIN (GUEST는 GNB 일부만 노출)
- **진입 경로 (URL)**: `/`
- **주요 컴포넌트 목록**:
  - `<GNBHeader>` - 글로벌 네비게이션 바 (로고, GNB 메뉴, 사용자 정보, 로그아웃)
  - `<LNBSidebar>` - 좌측 사이드 메뉴 (GNB 선택에 따라 동적 렌더링)
  - `<MainContent>` - 우측 메인 컨텐츠 영역 (라우터 Outlet)
  - `<BreadcrumbBar>` - 현재 위치 표시
  - `<UserBadge>` - 사용자명 + Role 뱃지

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 페이지 진입 | JWT 토큰 검증 → 유효 시 사용자 Role 기반 메뉴 목록 API 호출 |
| GNB 메뉴 클릭 | 해당 GNB에 속한 LNB 목록 렌더링, URL 변경 |
| LNB 메뉴 클릭 | 해당 페이지 라우팅, 브레드크럼 갱신 |
| 로그아웃 클릭 | `/api/auth/logout` 호출 → localStorage 토큰 삭제 → `/login` 리다이렉트 |
| 토큰 만료 감지 | 자동 로그아웃 + Toast 알림: "세션이 만료되었습니다" |
| 권한 없는 메뉴 진입 | `/403` 리다이렉트 |

## SB-002. 로그인 화면

- **화면 설명**: 포털 진입 인증 화면. Supabase Auth 기반 이메일/패스워드 로그인. 로그인 성공 시 JWT 발급 후 메인으로 이동.
- **접근 가능 Role**: GUEST (비로그인 상태)
- **진입 경로 (URL)**: `/login`
- **주요 컴포넌트 목록**:
  - `<LoginForm>` - 이메일 · 패스워드 입력 폼
  - `<LoginButton>` - 로그인 버튼 (로딩 스피너 포함)
  - `<ErrorMessage>` - 인증 오류 메시지 표시
  - `<PasswordToggle>` - 패스워드 표시/숨김 토글

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 이메일 입력 | 실시간 이메일 형식 유효성 검사 |
| 패스워드 입력 | 입력 마스킹, 최소 8자 체크 |
| 로그인 버튼 클릭 | `/api/auth/login` POST → JWT 수신 → localStorage 저장 → `/` 리다이렉트 |
| 이미 로그인 상태 진입 | 자동으로 `/` 리다이렉트 |
| 잘못된 인증 정보 | 인라인 에러: "이메일 또는 패스워드가 올바르지 않습니다" |
| 5회 실패 | 계정 잠금 안내 메시지 + 30초 재시도 제한 |
| Enter 키 입력 | 로그인 폼 제출 |

## SB-003. GNB 메뉴 관리 목록

- **화면 설명**: 포털 상단 GNB(Global Navigation Bar)에 표시될 1단계 메뉴를 조회·관리하는 화면. 메뉴 활성/비활성 토글, 등록/수정/삭제 기능 제공.
- **접근 가능 Role**: ADMIN, SUPER_ADMIN
- **진입 경로 (URL)**: `/admin/menu/gnb`
- **주요 컴포넌트 목록**:
  - `<PageHeader>` - 제목 + [GNB 등록] 버튼
  - `<GNBMenuTable>` - GNB 목록 테이블 (순서, 메뉴명, URL, 아이콘, 활성여부, 권한, 작업)
  - `<StatusToggle>` - 활성/비활성 스위치
  - `<ActionButtons>` - 수정/삭제 버튼
  - `<MenuFormModal>` - 등록/수정 모달 (SB-005 참조)
  - `<DeleteConfirmDialog>` - 삭제 확인 다이얼로그

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 페이지 진입 | `GET /api/menus?type=gnb` 호출 → GNB 목록 테이블 렌더링 |
| [GNB 등록] 클릭 | SB-005 등록 모달 오픈 (type=GNB, 신규) |
| 활성/비활성 토글 | `PATCH /api/menus/{id}` → 상태 즉시 반영 + Toast "상태가 변경되었습니다" |
| [수정] 클릭 | SB-005 수정 모달 오픈 (기존 데이터 채워진 상태) |
| [삭제] 클릭 | 확인 다이얼로그 출력: "하위 LNB가 모두 삭제됩니다. 계속하시겠습니까?" |
| 삭제 확인 | `DELETE /api/menus/{id}` → 목록 갱신 + Toast "삭제되었습니다" |
| [순서변경] 클릭 | SB-006 페이지 이동 |

## SB-004. LNB 메뉴 관리 목록

- **화면 설명**: GNB별 하위 LNB(Local Navigation Bar) 2depth 메뉴를 관리하는 화면. 상단에 GNB 선택 탭 제공, 선택된 GNB에 속한 LNB 목록 표시.
- **접근 가능 Role**: ADMIN, SUPER_ADMIN
- **진입 경로 (URL)**: `/admin/menu/lnb`
- **주요 컴포넌트 목록**:
  - `<GNBSelectTab>` - GNB 선택 탭 바
  - `<LNBMenuTable>` - LNB 목록 테이블 (depth, 부모메뉴, 메뉴명, URL, 순서, 활성여부, 작업)
  - `<DepthBadge>` - depth 1/2 구분 뱃지
  - `<ParentMenuBadge>` - 상위 메뉴명 표시
  - `<MenuFormModal>` - 등록/수정 모달 (SB-005 참조)

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 페이지 진입 | GNB 목록 탭 로드 → 첫 번째 GNB 자동 선택 → LNB 목록 표시 |
| GNB 탭 변경 | `GET /api/menus?type=lnb&gnbId={id}` 재호출 → LNB 목록 갱신 |
| [LNB 등록] 클릭 | SB-005 등록 모달 오픈 (type=LNB, 선택된 GNB ID 전달) |
| 활성 토글 | `PATCH /api/menus/{id}` → 상태 즉시 반영 |
| [수정] 클릭 | SB-005 수정 모달 오픈 |
| [삭제] 클릭 | 확인 다이얼로그 → `DELETE /api/menus/{id}` |

## SB-013. 메뉴-Role 매핑 관리 (체크박스 매트릭스)

- **화면 설명**: 메뉴(행)와 Role(열)을 매트릭스 형태로 배치하여, 체크박스로 접근 권한을 일괄 설정하는 화면. 직관적인 시각적 매핑으로 권한 현황을 한 눈에 파악 가능.
- **접근 가능 Role**: ADMIN, SUPER_ADMIN
- **진입 경로 (URL)**: `/admin/roles/mapping`
- **주요 컴포넌트 목록**:
  - `<MenuTypeFilter>` - GNB / LNB 필터 탭
  - `<PermissionMatrix>` - 메뉴×Role 체크박스 매트릭스 테이블
  - `<BulkSelectHeader>` - 열(Role) 전체 선택/해제 체크박스
  - `<SaveButton>` - 변경사항 일괄 저장
  - `<ChangedCountBadge>` - 변경된 항목 수 표시

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 페이지 진입 | `GET /api/roles/mapping` 호출 → 매트릭스 렌더링 |
| 체크박스 on/off | 로컬 상태 변경 + 변경 카운트 갱신 |
| Role 열 전체 선택 | 해당 Role에 모든 메뉴 권한 일괄 체크 |
| Role 열 전체 해제 | 해당 Role의 모든 메뉴 권한 일괄 해제 |
| [저장] 클릭 | `PUT /api/roles/mapping` (전체 매트릭스 전송) → Toast |
| [초기화] 클릭 | 로컬 변경사항 취소, API 데이터로 복원 |

| 비저장 상태에서 이탈 | "저장하지 않은 변경사항 {N}건이 있습니다" 경고 |

## SB-014. 사용자 Role 부여/회수 관리

- **화면 설명**: 개별 사용자에게 Role을 부여하거나 회수하는 화면. 사용자 검색, 현재 Role 확인, Role 변경 기능 제공.
- **접근 가능 Role**: ADMIN, SUPER_ADMIN
- **진입 경로 (URL)**: `/admin/users/roles`
- **주요 컴포넌트 목록**:
  - `<UserSearchBar>` - 이름/이메일 검색 + Role 필터
  - `<UserRoleTable>` - 사용자 목록 (이름, 이메일, 부서, 현재 Role, 최근 로그인, 작업)
  - `<CurrentRoleBadge>` - 현재 Role 뱃지 (색상 구분)
  - `<RoleChangeDropdown>` - Role 변경 드롭다운 (인라인)
  - `<BulkRoleSelect>` - 체크박스 다중 선택 후 일괄 Role 변경
  - `<Pagination>` - 페이지네이션 (20건/페이지)

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 페이지 진입 | `GET /api/users?includeRole=true` 호출 → 사용자 목록 표시 |
| 이름/이메일 검색 | debounce 300ms 후 `/api/users?q={keyword}` 재조회 |
| Role 필터 변경 | 해당 Role 사용자만 표시 |
| Role 드롭다운 변경 | `PATCH /api/users/{id}/role` → Toast "Role이 변경되었습니다" |
| 다중 선택 후 [일괄 변경] | 확인 다이얼로그 → `PATCH /api/users/roles/bulk` |
| 자기 자신의 Role 변경 | 경고: "본인의 Role은 변경할 수 없습니다" |

## SB-015. 권한 오류 페이지 (403 Forbidden)

- **화면 설명**: 접근 권한이 없는 페이지에 진입했을 때 표시되는 에러 페이지. 사용자의 현재 Role과 요구되는 Role을 안내하고, 적절한 탈출 경로를 제공한다.
- **접근 가능 Role**: 모든 Role (권한 부족 상태에서 자동 노출)
- **진입 경로 (URL)**: `/403` (또는 권한 없는 URL 접근 시 리다이렉트)
- **주요 컴포넌트 목록**:
  - `<ErrorCode>` - 403 대형 표시
  - `<ErrorIllustration>` - 접근 금지 일러스트
  - `<ErrorMessage>` - 현재 Role + 필요 Role 안내 텍스트
  - `<ActionButtons>` - [이전 페이지로] [메인으로] 버튼
  - `<ContactInfo>` - 권한 신청 안내 (관리자 이메일 링크)

### 사용자 액션 → 시스템 반응

| 사용자 액션 | 시스템 반응 |
|------------|------------|
| 권한 없는 URL 직접 입력 | PrivateRoute에서 감지 → `/403` 리다이렉트 |
| 권한 없는 메뉴 클릭 | 동일하게 `/403` 리다이렉트 |
| [이전 페이지로] 클릭 | `navigate(-1)` 실행 |
| [메인으로] 클릭 | `/` 홈으로 이동 |
| [관리자에게 권한 요청] 클릭 | 관리자 이메일 mailto 링크 또는 권한 요청 모달 |
| 비로그인 상태에서 접근 | `/login?redirect={originalPath}` 리다이렉트 |
