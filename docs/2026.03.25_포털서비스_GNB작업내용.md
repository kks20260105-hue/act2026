## 6. DB CHECK 제약 조건 수정

### 문제
- `menu_depth IN (1, 2)` 제약으로 depth 3 이상 메뉴 등록 불가

### 해결
- 마이그레이션 SQL 파일 생성: `infra/sql/20260325_alter_menu_depth_constraint.sql`

```sql
ALTER TABLE public.tb_menu DROP CONSTRAINT IF EXISTS tb_menu_menu_depth_check;
ALTER TABLE public.tb_menu ADD CONSTRAINT tb_menu_menu_depth_check CHECK (menu_depth >= 1);
```

- 스키마 파일 2개 수정 (`portal_menu_role_schema.sql`, `init.sql`)
- Supabase SQL Editor에서 마이그레이션 실행 완료

- 아이콘 삭제, L1/L2/L3 태그 삭제
- 10depth까지 전체 트리 펼침
- GNB 헤더 표시 (현재 GNB명 자동 감지)

### 변경 내용 (LNBLayout.tsx)

| 항목 | 변경 내용 |
|------|---------|
| GNB 헤더 | 아이콘(`getIcon`) 제거 → 텍스트만 표시 |
| L레벨 태그 | `lColor`, `MenuLabel` 함수 제거 |
| buildItems | icon, MenuLabel 제거 → 순수 `m.menu_nm` |
| 전체 펼침 | `getAllParentKeys()` 추가 → `defaultOpenKeys`에 전달 |
| Dead code 정리 | `import * as Icons`, `getIcon()`, `getOpenKeys()` 제거 |

### 핵심 구현 코드

```tsx
/** 자식이 있는 모든 메뉴 URL 수집 (전체 펼침용) */
function getAllParentKeys(allMenus: MenuType[]): string[] {
  const parentIds = new Set(
    allMenus.filter((m) => m.parent_menu_id).map((m) => m.parent_menu_id!)
  );
  return allMenus
    .filter((m) => parentIds.has(m.menu_id))
    .map((m) => m.menu_url);
}

// defaultOpenKeys에 전체 펼침 적용
const allOpenKeys = useMemo(() => getAllParentKeys(subMenus), [subMenus]);

<Menu
  mode="inline"
  selectedKeys={[location.pathname]}
  defaultOpenKeys={allOpenKeys}
  items={menuItems}
/>
```
